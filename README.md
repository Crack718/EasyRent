# EasyRent

EasyRent — веб-приложение для аренды жилья на `Vanilla JavaScript` и `Firebase`.
Проект покрывает ключевые сценарии сервиса аренды: каталог объявлений, карточка жилья, бронирования, профиль пользователя, админ-панель и real-time обновления.

## Зачем нужен сайт

EasyRent можно использовать как:

- базу для MVP сервиса аренды жилья;
- учебный проект по Firebase (Auth, Firestore, Rules, Indexes);
- стартовый шаблон для дальнейшей коммерческой разработки.

## Как работает сайт

1. Пользователь открывает каталог на главной странице.
2. Использует поиск, фильтры и сортировку, затем догружает объявления кнопкой «Показать ещё».
3. Открывает карточку объявления, смотрит фото, отзывы, доступность и похожие варианты.
4. Может отправить сообщение владельцу, добавить в избранное и оформить бронирование.
5. В личном кабинете управляет профилем, отзывами, сообщениями и историей.
6. В админ-панели администратор управляет контентом и пользователями.

## Реализованные страницы

- `index.html` — каталог с real-time, поиском, фильтрами, сортировкой и пагинацией.
- `listing.html` — детальная страница объявления (галерея, отзывы, сообщения, бронирование).
- `profile.html` — профиль пользователя, избранное, история, мои отзывы и сообщения.
- `cart.html` — активные и завершённые бронирования.
- `bookings.html` — редирект на `cart.html`.
- `admin.html` — админ-панель с CRUD и модерацией.

## Основной функционал

- регистрация, вход, выход, восстановление пароля;
- роли `user` и `admin`;
- real-time обновления через `onSnapshot`;
- пагинация через `limit()` и `startAfter()`;
- поиск по названию/описанию;
- фильтрация по типу жилья, региону, району, удобствам, цене, комнатам, площади, рейтингу;
- сортировка по дате, цене, рейтингу;
- избранное;
- бронирования с проверкой пересечения дат;
- админ-управление объявлениями, пользователями, сообщениями и отзывами.

## Технологии

- `HTML5`, `CSS3`, `Vanilla JS (ES6 modules)`;
- `Firebase Authentication`;
- `Cloud Firestore`;
- `Firebase Security Rules`;
- `Firestore Indexes`;
- опционально: `Cloud Functions` (endpoint `catalogQuery`).

## Firestore: коллекции

- `users` — профили пользователей и роли.
- `listings` — полные объявления.
  - подколлекция `images`.
  - подколлекция `reviews`.
- `listing_cards` — облегчённые карточки каталога (денормализация для скорости).
- `amenities`, `regions`, `districts`, `property_types` — справочники.
- `favorites` — избранные объявления пользователя.
- `messages` — переписка пользователь ↔ владелец.
- `bookings` — все бронирования.
- `availability` — занятые слоты дат.
- `active_actions` — активные действия.
- `action_history` — завершённые/отменённые действия.

## Безопасность

Правила в `firestore.rules` реализуют:

- разграничение доступа по ролям `user/admin`;
- доступ к админским операциям только для `admin`;
- чтение/изменение только своих пользовательских данных;
- ограничения на редактирование сообщений и бронирований;
- публичное чтение каталога при ограниченной записи.

## Локальный запуск

### 1) Установка зависимостей

```powershell
npm install
```

### 2) Запуск статического сервера

```powershell
npx http-server . -p 5173
```

Открыть в браузере:

- [http://127.0.0.1:5173/index.html](http://127.0.0.1:5173/index.html)

## Настройка Firebase

1. Создайте или используйте проект Firebase.
2. Включите `Authentication` (Email/Password).
3. Создайте базу `Firestore`.
4. Проверьте `window.EASYRENT_CONFIG` в HTML-страницах.
